# HandoffBox

HandoffBox は、AI エージェントと人間が共有して読める公開プロジェクト台帳です。

このリポジトリは、個人プロジェクトにおける AI との共有コンテキストをまとめたも
のです。主に AI エージェント間の相互運用性を目的として公開しています。

URL だけで読める形で、プロジェクトの目的、設計思想、次にやること、決定済みの
判断を残します。アプリ本体のコード、個人情報、認証情報、顧客情報、非公開情報を
置く場所ではありません。

## 公開運用する理由

HandoffBox は、どの AI エージェントでも同じ文脈を読めるように公開リポジトリとし
て運用します。

非公開リポジトリでは、エージェントごとに GitHub の認証状態や権限が異なり、
「読めない」「権限がない」「アクセスできない」という問題が繰り返し発生します。
公開台帳にすることで、URL だけで文脈を共有できます。

その代わり、誰が読んでも問題ない情報だけを書くことを徹底します。

## 置いてよいもの

- プロジェクト概要
- 設計思想
- 次にやること
- 判断理由
- 公開しても問題ない作業ログ
- AI エージェント向けの引き継ぎメモ

各プロジェクトは `projects/{project-id}/` に置き、次の4ファイルで管理します。

- `PROJECT.md`: プロジェクトの目的、範囲、読者、公開リンク
- `NEXT.md`: 次にやる具体的なこと
- `JOURNAL.md`: 公開可能な作業ログ
- `DECISIONS.md`: 決定事項と理由

新しい台帳を作るときは `templates/` を使ってください。

## 絶対に書いてはいけないもの

このリポジトリには、以下を絶対に書かないでください。

- 個人情報
- メールアドレス
- 電話番号
- 住所
- API キー
- アクセストークン
- パスワード
- ローカル環境の絶対パス
- 収支や生活費などの個人的な金銭情報
- 顧客情報
- 未公開の仕入れ候補
- 個人情報を含むスクリーンショット
- 非公開リポジトリの機密情報

作業ログを書く場合は、公開しても問題ない抽象度に要約してください。非公開プロ
ジェクト内にだけ置くべき詳細は、HandoffBox には書かないでください。

## ディレクトリ構成

```text
HandoffBox/
├── README.md
├── AGENTS.md
├── projects/
│   └── example-project/
│       ├── PROJECT.md
│       ├── NEXT.md
│       ├── JOURNAL.md
│       └── DECISIONS.md
└── templates/
    ├── PROJECT.template.md
    ├── NEXT.template.md
    ├── JOURNAL.template.md
    └── DECISIONS.template.md
```

## 運用方針

HandoffBox はタスク管理ツールではありません。AI エージェントが読むための共有文脈
置き場です。

残すべきものは、実装そのものではなく、次のような文脈です。

- なぜ作るのか
- 今の計画はどうなっているのか
- 次のエージェントは最初に何をすべきか
- どの判断がすでに決まっているのか
- 現在の状態を支える公開可能な根拠は何か

アプリ本体のコード、非公開情報、認証情報、機密ログは、それぞれ適切なリポジトリ
や管理システムに置いてください。
