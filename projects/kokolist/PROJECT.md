# Koko List

## Summary

Koko List is a device-first PWA for place-based reminders.

## Purpose

- Help users remember tasks, purchases, questions, and errands because of where they are.
- Show the relevant list immediately when the user opens the app at a registered place.
- Keep the product calm, private, practical, and fast.

## Audience

- Primary users: Busy individuals who want fewer forgotten errands and location-specific tasks.
- Secondary users: Unknown / 不明

## Scope

In scope:

- Registering places.
- Registering a place from the current location.
- Registering a place by search or manual input.
- Attaching reminder/checklist items to each place.
- Detecting the current location when the app opens.
- Showing the matching place list first when a registered place matches.
- Supporting local, on-device storage first.

Out of scope:

- Native geofence notifications for the first MVP.
- Push notifications for the first MVP.
- User accounts and cloud sync for the first MVP.
- Family or team sharing.
- Public place database or community features.
- Heavy map-first browsing.

## Agent Notes

- Treat the app as a place-based reminder product, not only a shopping list.
- Preserve the device-first privacy position unless a human explicitly changes the product direction.
- Do not promise automatic arrival notifications in the MVP; that belongs to a later native capability decision.
